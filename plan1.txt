# AI Service Refactoring Plan

## Overview

Refactor `includes/services/class-cdw-ai-service.php` from a monolithic 2,213-line class into a modular handler-based architecture.

**Target**: Reduce main file from 2,213 lines to ~400 lines (82% reduction)

---

The current `class-cdw-ai-service.php` contains 12 public static methods:

| Method | Lines | Purpose |
|--------|-------|---------|
| `get_providers()` | ~65 | Returns provider catalogue |
| `encrypt_api_key()` | ~25 | Encrypts API keys |
| `decrypt_api_key()` | ~55 | Decrypts API keys |

| `save_user_ai_settings()` | ~50 | Saves user AI settings |
| `get_decrypted_api_key()` | ~20 | Gets decrypted API key |
| `check_ai_rate_limit()` | ~30 | Rate limiting |
| `get_tool_definitions()` | ~850 | Returns 40+ tool definitions (HUGE) |
| `build_system_prompt()` | ~50 | Builds system prompt |
| `execute_tool_call()` | ~110 | Executes tool calls |
| `execute_agentic_loop()` | ~220 | Main orchestrator |
| `test_api_key()` | ~20 | Tests API connection |

Plus private methods:
- `call_openai()` (~60 lines)
- `call_anthropic()` (~50 lines)
- `call_google()` (~50 lines)
- `tool_name_to_cli_command()` (~100 lines)
- Various helper methods

```
includes/services/ai/

└── class-cdw-ai-prompts.php        # System prompts (~50 lines) - EXTRACT
```

## Extraction Details

### 1. class-cdw-ai-tools.php (~900 lines)

Extract these methods from `class-cdw-ai-service.php`:

**get_tool_definitions()** (lines ~396-1245, ~850 lines)
- Returns array of 40+ tool definitions for AI function calling
- Each tool has name, description, parameters

**tool_name_to_cli_command()** (lines ~1414-1523, ~110 lines)
- Maps tool names to CLI command strings
- Switch statement with 40+ cases

**execute_tool_call()** (lines ~1296-1405, ~110 lines)
- Executes tool calls from AI providers
- Handles special cases (gutenberg_guide, post_set_content, post_create, page_create)
- Delegates to CDW_CLI_Service for most commands

### 2. class-cdw-ai-providers.php (~500 lines)

Extract these private static methods:

**call_openai()** (lines ~1634-1690, ~60 lines)
- Makes API calls to OpenAI-compatible endpoints
- Handles response parsing, errors, usage stats

**call_anthropic()** (lines ~1692-1741, ~50 lines)
- Makes API calls to Anthropic Claude
- Different message format than OpenAI

**call_google()** (lines ~1743-1791, ~50 lines)
- Makes API calls to Google Gemini
- Different message format and tool handling

**Helper methods** (format_assistant_tool_call_message, format_tool_result_message, etc.)
- Used by all provider methods

### 3. class-cdw-ai-prompts.php (~50 lines)

Extract:

**build_system_prompt()** (lines ~1245-1295, ~50 lines)
- Builds the system prompt that instructs the AI on capabilities

## Key Decisions

1. **Extract incrementally** - Do one file at a time, test after each
2. **Keep method signatures identical** - Don't change method signatures, just delegate
3. **Static methods** - All methods remain static (same as original)
4. **Test after each extraction** - Don't skip testing
5. **No functional changes** - Copy/paste only, don't refactor logic
